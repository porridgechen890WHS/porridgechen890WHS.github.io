cmake添加变量

set (CMAKE_VAR_1 666)这样就相当于设置了一个变量CMAKE_VAR_1，初始化为666。
message("CMAKE_VAR_1=${CMAKE_VAR_1}")这样就可以打印这个变量的值。

cmake添加配置文件

  • 为什么要加配置文件？
    假如你想要给你的工程设置一个版本号，当然你可以写在代码里，不过写在CMakeLists.txt里不是更好吗？
    假设你在CMakeLists.txt里定义一个变量CMAKE_VAR_1用来表示版本号。
    set(CMAKE_VAR_1 666)这句话我是写在project()前面的。

  • 第一步：新建一个配置文件
    你自己新建一个配置文件，假设叫A.h.in。
    就写入#define PROJ_VER @CMAKE_VAR_1@，保存到跟CMakeLists.txt同级的目录，当然你也可以放到其他目录。

  • 第二步：修改CMakeLists.txt，加载那个配置文件，并指定输出的文件名
    configure_file(A.h.in B.h)
    这句话的意思是告诉cmake，你把那个叫做A.h.in的文件当作配置文件，并输出成B.h供代码里使用。
    我是把这句话放在project()这条命令后面的。或许放在别处也可以呢。

  • 第三步：修改CMakeLists.txt，把配置文件所在的目录添加到头文件搜索目录
    target_include_directories(PROJ_NAME PUBLIC "${PROJECT_BINARY_DIR}" )
    因为配置文件将会被写入到二进制树当中，我们需要把这个目录添加到头文件路径当中。
    我是把这句话放在CMakeLists.txt末尾的。或许放在别处也可以呢。

  • 第四步：在代码里使用配置文件
    #include "B.h"
    fprintf(stdout, "PROJ_VER=%d\n", PROJ_VER);
